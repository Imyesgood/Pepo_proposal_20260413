# 엑셀 파일 관리 가이드

## 파일 위치
```
data/raw/raw.xlsx    ← 여기에 넣기
```

---

## 시트 이름 규칙

| 시트 이름 형식 | 예시 | 역할 |
|---|---|---|
| `SECTOR_{등급}{한글명}` | `SECTOR_AAA은행채` | 섹터별 일별 수익률 시계열 |
| `SECTOR_{등급}{한글명}` | `SECTOR_AA-기타금융채` | 동일 |
| `ISSUER(BOND)` | 고정값 | 발행사별 현재 만기별 수익률 |
| `ISSUER(CD)` | 고정값 | 발행사별 CD 금리 |
| `기준금리` | 고정값 | 한국은행 기준금리 시계열 |
| `PLOT_*`, `Sheet*` 등 | 아무거나 | **자동 무시됨** |

### 등급 표기 규칙
```
AAA, AA+, AA-, AA (공백 없이)
예: SECTOR_AAA은행채  SECTOR_AA+공사채  SECTOR_AA-기타금융채
```

## SECTOR 시트 내부 구조

```
Row 0: ∼  [블룸버그 헤더명]  NaN  NaN  ...  ∼  [블룸버그 헤더명2]  NaN ...
Row 1: 날짜  3M  6M  9M  1Y  날짜  1.5Y  2Y  2.5Y  ...       ← 헤더 (자동 감지)
Row 2: 2026-04-08  2.791  2.864  ...  2026-04-08  3.331  ...  ← 데이터
Row 3: 2026-04-07  2.801  ...
...
(마지막 쪽에 빈행, "평  균" 행, 주석 행 있어도 자동 무시됨)
```

**핵심 규칙:**
- Row 0은 완전 자유 (블룸버그 쓰레기값 있어도 됨)
- **`날짜`** 라는 단어가 있는 행이 헤더로 자동 인식됨
- 날짜-만기 블록이 좌→우로 여러 개 반복되어도 자동 병합
- 만기 표기: `3M`, `6M`, `1Y`, `1Y6M`, `2Y`, `2Y6M`, `3Y` ... (블룸버그 형식 그대로)

## ISSUER(BOND) 시트 내부 구조

```
Row 0: 회사코드  회사명  평가사  분류 ▼  3M  6M  9M  1Y  1Y6M  2Y  ...  ← 헤더 (자동 감지)
Row 1: 00001    신한은행   3사   금융채 은행채 AAA   2.791  2.864  ...
Row 2: 00494    하나은행   3사   금융채 은행채 AAA   2.791  2.864  ...
...
```

**핵심 규칙:**
- **`회사코드`** 라는 단어가 있는 행이 헤더로 자동 인식됨
- 분류 컬럼명은 `분류`, `분류 ▼` 등 어느 것이든 자동 인식
- 만기 컬럼 자동 감지 (변환 가능한 것만: `3M`~`30Y`)
- 스프레드 컬럼(`1Y6M-1Y` 등)은 자동 무시

**분류 컬럼 표기 → 시스템 인식:**
| 분류 값 | 인식 결과 |
|---|---|
| `금융채 은행채 AAA` | 은행채(AAA) |
| `금융채 은행채 AA+` | 은행채(AA+) |
| `공사/공단채 AAA` | 공사채(AAA) |
| `공사/공단채 AA+` | 공사채(AA+) |
| `공사/공단채 AA0` | 공사채(AA) |
| `공사/공단채 정부보증` | 공사채(AAA, 정부보증) |
| `금융채 기타금융채 AA-` | 기타금융채(AA-) |
| 기타 | 자동 파싱 시도 |

## ISSUER(CD) 시트 내부 구조

```
Row 0: 기관코드  회사명  평가사  신용등급  1D  7D  10D  15D  1M  2M  3M  6M  1Y  2Y
Row 1: 00001    신한은행   3사    AAA      2.56  2.6  ...
...
```

**핵심 규칙:**
- **`기관코드`** 라는 단어가 있는 행이 헤더로 자동 인식
- 만기 컬럼 자동 감지 (`1D`, `7D`, `1M`, `3M`, `1Y` 등)

---

## 기준금리 시트 내부 구조

```
Row 0: 한국:기준금리  단위: %          ← 무시됨
Row 1: 일자          현재가            ← 헤더 (자동 감지)
Row 2: 2026-04-14   2.5
Row 3: 2026-04-13   2.5
...
```

**핵심 규칙:**
- **`일자`** 라는 단어가 있는 행이 헤더로 자동 인식
- 값은 % 단위 (2.5 → 내부적으로 0.025로 변환)
- NaN 행(주말 등) 자동 제거

## 자주 묻는 것

**Q: Row 0에 블룸버그 헤더가 있어도 되나요?**
A: 됩니다. Row 0은 무조건 무시됩니다. `날짜`, `회사코드`, `기관코드`, `일자` 키워드로 헤더 위치를 자동 감지합니다.

**Q: SECTOR 시트에서 단기/장기 블록이 두 개로 나뉘어 있어도 되나요?**
A: 됩니다. `날짜` 컬럼이 여러 개면 블록 경계로 자동 인식하고 합칩니다.

**Q: 하단에 "평  균" 행이나 주석 행이 있어도 되나요?**
A: 됩니다. 날짜가 datetime 타입인 행만 데이터로 인식하고, 나머지는 자동 제거됩니다.

**Q: 시트를 추가/삭제하면?**
A: 시트 이름 규칙만 맞으면 자동으로 읽습니다. `PLOT_*`, `Sheet*` 같은 이름은 자동 무시됩니다.

**Q: ISSUER(BOND)에 새 섹터/등급이 추가되면?**
A: 자동으로 드롭다운에 반영됩니다. 하드코딩 없이 파일에 있는 것만 나옵니다.
